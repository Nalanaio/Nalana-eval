## 1. What this is

ADR-005 documents the schema-level taxonomy decisions for #13 (Test case authoring pipeline): drop the conflated `Difficulty` axis, add `SceneComplexity` (manually authored), evaluate spatial coherence on COMPOSITION cases via the L3 judge instead of hard `relative_positions` constraints. The ADR is the design foundation for #13.1 (schema changes) and #13.2 (existing fixture re-audit).

This PR is **doc-only** — no code changes. The actual schema migrations land in #13.1's PR; the existing-fixture re-audit lands in #13.2.

## 2. Why now — what triggered this

Two converging signals from PR-A end-to-end verification (2026-04-29) and the Q3 prototype experiment (Task #22, 2026-04-30):

- **PR-A audit** found `Difficulty` labels are wrong/inconsistent on multiple existing fixtures (e.g., CV-AMB-004 "Make a simple house" labeled `Long` but the expected scene is just cube + cone). The original authoring intent of `Difficulty` was "step count" — the same v2 ground-truth-replication mental model V3 explicitly abandoned in ADR-001's predecessor decision.

- **Task #22 prototype** tested whether the L3 judge can distinguish coherent vs incoherent compositional outputs. Result: **yes on non-empty scenes** (semantic=1.0 correctly assigned to a single cube tagged "doesn't resemble a table"), **no on empty scenes** — judge hallucinates full table descriptions when shown a fallback PNG. The empty-scene guard hotfix (`ian/judge-empty-scene-guard`) addresses that and is a hard prerequisite for #13.2.

These two together motivated formalizing the taxonomy + spatial-coherence-evaluation strategy as an ADR before #13.1 / #13.2 implementation begins, so the chain of decisions is documented and reviewable in one place.

## 4. Decisions made

Five questions resolved in chat 2026-04-30; full details in ADR-005 itself.

```
Q1. Difficulty: keep deprecated as Optional one cycle → removal in v3.2.
Q2. TaskLength: drop entirely. Histogram showed all 80 existing prompts clustered
    in 3-9 words; axis would be a constant column. Original intent ("step count") was
    v2 ground-truth model = abandoned in V3.
Q3. COMPOSITION boundary: subjective (author intent), evaluated via L3 judge,
    NOT hard relative_positions. Task #22 prototype confirmed judge can distinguish
    on non-empty scenes (and ian/judge-empty-scene-guard handles the empty-scene case).
Q4. scene_complexity: manually authored. Not auto-derived from constraint shape;
    decoupling preserves author intent and enables drift_check cross-validation.
Q5. #13.2 audit operates in strict mode: tag + flip judge_policy=score + expand
    acceptable_styles for COMPOSITION/FULL_SCENE cases, one pass per case-file batch.
    Cost rises ~3.5×, accepted.
```
